# Support

## Product and development questions

Use repository Discussions when enabled. Until then, open a focused issue only when the repository's issue forms provide an appropriate category.

## Bugs

Use the bug-report issue form in the affected repository. Include reproduction steps, expected/actual behavior, environment, and a minimal example. Remove secrets and private source.

## Features and RFCs

Use the feature or RFC issue form. Large or breaking changes require an RFC before implementation.

## Security

Follow [SECURITY.md](./SECURITY.md). Never report vulnerabilities publicly.

## Abuse, ownership, license, and takedown concerns

Do not publish private claimant information or proprietary source in a public issue. Until a dedicated trust-and-safety channel launches, use the [private organization advisory form](https://github.com/Modulora/.github/security/advisories/new) and identify the report as an abuse, ownership, license, or takedown concern.

## Commercial support

No paid support or SLA is currently offered. Any future Cloud or Enterprise support terms will be documented separately.
